# GioGia PRD v1

## Product Name

GioGia

## Vision

GioGia is an AI-powered web testing platform that combines semantic understanding, exploratory testing, and deterministic test execution.

The platform enables users to:

* Discover application workflows using AI
* Capture semantic snapshots of web applications
* Generate tests from natural language specifications
* Execute tests deterministically
* Produce actionable reports
* Support manual coding, BDD, and Page Object Models (POM)

AI is primarily used during discovery, authoring, and maintenance phases, while execution remains deterministic and CI/CD friendly.

---

# Problem Statement

Current testing tools suffer from several limitations:

1. Locators are brittle and break frequently.
2. Test creation requires significant manual effort.
3. Exploratory testing is difficult to scale.
4. Generated tests become outdated quickly.
5. Non-technical stakeholders struggle to contribute to test design.
6. AI-based testing tools often produce non-deterministic execution results.

GioGia addresses these issues through semantic snapshots, AI-assisted discovery, and deterministic execution.

# Core Principles

## AI for Authoring

AI should help users:

* Discover workflows
* Explore applications
* Generate tests
* Suggest improvements
* Heal locators

## Deterministic Execution

Runtime execution must:

* Be repeatable
* Produce consistent results
* Avoid runtime AI decision making
* Work offline when possible

## Human Editable

Anything generated by AI must remain editable by humans.

## Escape Hatch

Advanced users must be able to write code directly.

## Snapshot

```bash
giogia snapshot <url>
```

Generate semantic snapshot.

Purpose:

* Capture application structure
* Store semantic locators
* Build application knowledge base

Generated artifact:

```text

```

Example:

```json
{
  "id": "login_button",
  "role": "button",
  "name": "Login"
}
```

# Semantic Locator System

## Goal

Provide stable locators independent of DOM structure.

Example:

```ts
await gio.click("checkout_button");
```

Instead of:

```ts
page.locator(
  "#app > div:nth-child(5)"
);
```

---

## Resolution Strategy

1. Semantic ID
2. Role
3. Label
4. Text
5. Test ID
6. CSS
7. XPath

Future:

8. Vision fallback
